# View Project Constitution

Read and display the project constitutional spec at `.specs/constitution.md`.

This document contains:
- Core principles (progressive complexity, platform simplification, AI-assisted evolution)
- Current architectural state and phases for all domains
- Quality standards (testing, security, performance, code quality)
- Technology stack (approved and forbidden patterns)
- Development workflows
- Cost constraints and Firebase free tier limits
- References to foundational guidance in `.prompts/`

**Usage**: Use this command to:
- Check project-specific constraints before starting a feature
- Verify current architectural phase for a domain
- Review quality standards and targets
- Understand forbidden patterns to avoid
- Find references to detailed patterns in `.prompts/`
